# National Critical Information Infrastructure (NCII) Security Assessment, Threat Analysis, and Data Protection Compliance Review

## Project Overview

This repository contains the final submission for Project 2 of the Ethical Hacking Professional Course under CYBERGON Cybersecurity Training.

The project focused on conducting a comprehensive National Critical Information Infrastructure (NCII) security assessment, threat analysis, vulnerability review, and data protection compliance evaluation across multiple sectors and states using industry-recognized cybersecurity frameworks and methodologies.

The assessment was performed using publicly available information, passive reconnaissance techniques, vulnerability intelligence resources, threat modeling methodologies, and compliance-oriented security analysis.

## Project Objective

The objective of this project was to simulate the activities of a National Cybersecurity Auditing Unit responsible for assessing critical infrastructure organizations and evaluating their security posture, cyber risks, compliance obligations, and potential threat exposure.

The assessment aligned with:

* Nigeria Data Protection Act (NDPA)
* NIST Cybersecurity Framework (CSF)
* MITRE ATT&CK Framework
* CVE Program
* CWE Catalog
* CVSS Scoring Framework
* EPSS Threat Prioritization
* OWASP Top 10

---

## Project Requirements

The project required each team to:

* Assess assigned sectors across multiple states
* Conduct OSINT and reconnaissance activities
* Map exposed attack surfaces
* Identify security weaknesses
* Analyze technical and operational risks
* Map findings to CVE, CWE, and OWASP categories
* Assess exploitability using CVSS and EPSS
* Develop mitigation strategies
* Recommend long-term remediation plans
* Evaluate compliance implications under NDPA

## Security Frameworks Used

### MITRE ATT&CK

Used to understand attacker behaviors, tactics, techniques, and procedures associated with identified weaknesses.

### CVE

Used to identify publicly disclosed vulnerabilities affecting exposed technologies and services.

### CWE

Used to classify software and security weaknesses according to industry standards.

### CVSS

Used to calculate vulnerability severity and business risk.

### EPSS

Used to estimate the probability of real-world exploitation.

### OWASP Top 10

Used to classify web application security weaknesses.

### NIST Cybersecurity Framework

Used as the primary framework for security assessment and control recommendations.

### Nigeria Data Protection Act (NDPA)

Used to evaluate data protection obligations and privacy-related risks.
